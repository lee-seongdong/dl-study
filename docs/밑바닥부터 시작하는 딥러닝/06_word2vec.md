# word2vec
단어를 고정 크기의 밀집 벡터(분산 표현)로 변환하는 추론 기반 기법  
주변 단어(맥락)가 주어졌을 때 특정 위치에 어떤 단어가 들어갈지 추론하는 방식으로 학습  
학습된 가중치(은닉층의 W_in)가 곧 단어의 분산 표현이 됨 (W_in이 여러개 인 경우 평균)

## 통계 기반 vs 추론 기반

| | 통계 기반 (SVD) | 추론 기반 (word2vec) |
|---|---|---|
| 학습 방식 | 전체 데이터 한꺼번에 처리 | 미니배치 반복 학습 |
| 시간복잡도 | O(n^3) | O(n) |
| 데이터 추가 시 | 전체 재학습 필요 | 추가 학습 가능 |

## CBOW (Continuous Bag of Words)
맥락(주변 단어들) → 타깃(중심 단어) 예측

```
context[t-1], context[t+1]  →  [hidden layer]  →  target[t]
```

- 입력층이 맥락 단어 수만큼 존재, 은닉층에서 평균을 취해 다음 계층으로 전달
- 학습 완료 후 W_in의 각 행이 각 단어의 분산 표현

## Skip-gram
타깃(중심 단어) → 맥락(주변 단어들) 예측 (CBOW의 반대 방향)

```
target[t]  →  [hidden layer]  →  context[t-1], context[t+1]
```

- 일반적으로 CBOW보다 단어 분산 표현의 품질이 높음
- 학습 데이터가 적거나 희귀 단어가 많을 때 유리

## 단순 CBOW의 문제점과 개선 기법

### 문제점
출력층에서 vocab 전체(V개)에 대해 softmax를 계산 → vocab이 클수록 계산량 O(V)

### 1. Embedding 레이어
one-hot 벡터 × W 행렬 곱은 결국 W의 특정 행을 꺼내는 것과 동일  
→ `W[word_id]` 인덱싱으로 대체해 one-hot 배열 생성 비용 제거

### 2. Negative Sampling (네거티브 샘플링)
다중 분류 문제를 이진 분류 문제로 변환

- **기존**: V개 전체에 대해 softmax → cross-entropy
- **개선**: 정답 1개 + 무작위 오답 k개에 대해 sigmoid → binary cross-entropy

```
(context, 정답 단어)  → sigmoid → 1 에 가깝게 학습
(context, 오답 단어1) → sigmoid → 0 에 가깝게 학습
(context, 오답 단어k) → sigmoid → 0 에 가깝게 학습
```

계산량: O(V) → O(k), k는 보통 5~20  
오답 단어는 완전 무작위가 아닌 **등장 빈도의 0.75승에 비례한 확률**로 샘플링

### 3. Subsampling
고빈도 단어(the, a 등)를 확률적으로 학습에서 제외  
→ 학습 효율 향상, 저빈도 단어의 학습 기회 증가

## 단어의 분산 표현 활용

학습된 분산 표현(단어 벡터)은 의미적/문법적 관계를 벡터 연산으로 표현할 수 있음

- 유사 단어 검색: 코사인 유사도로 가장 가까운 단어 탐색
- 유추 문제: `king - man + woman ≈ queen`

## 전이 학습 (Transfer Learning)
대규모 코퍼스로 사전 학습된 단어 벡터를 다른 태스크에 재사용하는 기법

- word2vec, GloVe 등의 사전 학습 임베딩을 그대로 활용
- 적은 데이터로도 좋은 성능을 낼 수 있음
- 방식 1: 임베딩 가중치를 **고정(freeze)** 하고 이후 계층만 학습
- 방식 2: 임베딩 가중치도 **fine-tuning** (태스크에 맞게 미세 조정)
