# 가상환경

## 1. 종류
- [**venv**](venv.md)
- [**uv**](uv.md)

## 2. 차이
| 작업 | venv + pip | uv |
|------|-----------|-----|
| 가상환경 생성 | `python -m venv .venv` | `uv venv` 또는 자동 생성 |
| 활성화 | `source .venv/bin/activate` | 불필요 (`uv run` 사용) |
| 패키지 설치 | `pip install requests` | `uv add requests` |
| 의존성 기록 | `pip freeze > requirements.txt` (수동) | 자동 (`pyproject.toml`) |
| 의존성 설치 | `pip install -r requirements.txt` | `uv sync` |
| 패키지 목록 | `pip list` | `uv pip list` 또는 `uv tree` |
| 실행 | `python main.py` (활성화 필요) | `uv run python main.py` |

## 3. 마이그레이션 (venv -> uv)
```bash
uv init --no-workspace

uv venv myenv

# requirements에 있는 패키지를 uv로 추가
cat requirements.txt | xargs uv add

uv run python main.py
```
